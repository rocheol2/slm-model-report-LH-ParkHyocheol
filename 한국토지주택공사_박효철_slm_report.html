01모델 선택 이유

한국토지주택공사(LH)의 방대한 사내 규정, 사업 지침서 및 PPTX 형식의 보고서를 기반으로 하는 RAG(검색 증강 생성) 시스템 구축에 최적화된 성능을 제공하기 때문입니다. Qwen2.5-7B-Instruct 모델은 최대 128K 토큰(131,072 토큰)의 매우 긴 문맥(Context Length)을 처리할 수 있어 다수의 문서를 한 번에 참조해야 하는 실무 환경에 적합하며, 구조화된 데이터(표, JSON 등)의 이해도가 우수하다는 Hugging Face 공식 모델 카드의 명시적 근거를 바탕으로 선정하였습니다.

03학습 데이터와 튜닝 방식

Hugging Face 공식 모델 카드에 따르면, 본 모델은 'Pretraining & Post-training' 단계를 거쳤습니다. 특화된 전문가 모델의 이점을 활용해 코딩 및 수학 능력이 향상되었으며, 지시 이행(Instruction Following) 능력을 높이기 위해 Instruction-tuned 되었습니다(모델명의 '-Instruct' 접미사가 이를 증명). Qwen 시리즈 공식 기술 문서(arXiv 2407.10671) 및 GitHub 문서에 따르면, 사전학습 이후 지도 학습 파인튜닝(SFT)과 직접 선호도 최적화(DPO, Direct Preference Optimization) 기법이 적용된 것으로 공개되어 있습니다. 다만, 사전 학습 데이터셋의 세부 구성 출처나 데이터 비율 등의 상세 정보는 모델 카드 내에 공개되어 있지 않습니다.

04파라미터 및 구조적 특징

ArchitectureTransformer with RoPE, SwiGLU, RMSNorm, Attention QKV bias
Context Length128K 토큰 (131,072 토큰) · 기본 config: 32,768 / YaRN 기법으로 확장 · 최대 생성 길이: 8,192 토큰
Hidden Size3,584 (config.json num_hidden_layers 기준)
Layers28 (num_hidden_layers)
Attention HeadsGQA 적용 (Q를 위한 헤드 28개, KV를 위한 헤드 4개)
TokenizerTiktoken 기반 BPE · Vocabulary 크기: 152,064 토큰 (config.json vocab_size 기준)

05모델 파일 구성

아래는 Hugging Face 공식 저장소 (tree/main)에서 직접 확인한 실제 파일 목록과 각 파일의 역할입니다. 총 저장소 크기는 약 15.2 GB입니다.

1. config.json (825 B): 모델 아키텍처(레이어 수, 어텐션 헤드, hidden size 등) 및 하이퍼파라미터를 정의하는 핵심 설정 파일입니다.
2. generation_config.json (243 B): top_p, temperature, max_new_tokens 등 텍스트 생성 시 사용되는 기본 파라미터를 제어합니다.
3. model-00001~00004-of-00004.safetensors (각 3.56~3.95 GB, 합계 약 15.2 GB): 모델의 실제 가중치(Weights) 데이터를 4개 샤드(Shard)로 분할 저장한 파일입니다. 기존 .pkl/.bin 포맷 대비 악성 코드 실행 위험을 차단한 안전한 Safetensors 포맷을 채택하였습니다.
4. model.safetensors.index.json (27.8 kB): 4개로 분할된 가중치 샤드 파일 간의 레이어 매핑 정보를 담고 있어, 모델 로딩 시 각 파라미터가 어느 샤드에 위치하는지 참조하는 인덱스 파일입니다.
5. merges.txt (1.67 MB): BPE(Byte-Pair Encoding) 토크나이저의 병합 규칙이 기록된 파일로, 텍스트를 서브워드 단위 토큰으로 분해하는 기준을 정의합니다.
6. sparse_attention_config.json: 1M 컨텍스트 처리를 위한 Sparse Attention(희소 어텐션) 설정 파일로, 1M 버전 특유의 장문 추론 최적화 구성 정보를 담고 있습니다.
7. LICENSE / README.md / .gitattributes: Apache-2.0 라이선스 원문, 모델 사용 안내 문서, Git LFS 대용량 파일 관리 설정 파일입니다.

06활용 가능 업무

한국토지주택공사 내부에 축적된 방대한 규정, 사업 지침서, 그리고 PPTX 형태의 정책 보고서 등을 기반으로 실무자의 질의에 신속하고 정확하게 답변하는 '내부 문서 기반 RAG QA 시스템' 구축 업무에 도입할 수 있습니다. 128K 규모의 Context Window를 통해 수십 페이지 분량의 문서를 청크(Chunk) 단위로 과도하게 쪼개지 않고 원문 전체 문맥을 유지한 채 검색 및 요약 업무를 수행할 수 있습니다.

07한계와 주의사항

텍스트와 구조화된 데이터(JSON, 마크다운 표 등)에 대한 이해도는 뛰어나나, PPTX 파일 내부에 존재하는 복잡한 시각적 다이어그램, 이미지 텍스트, 비정형 그래프 등을 순수 텍스트로 파싱하여 입력할 경우 '문맥 유실'이 발생할 기술적 위험이 존재합니다. 또한 긴 문맥을 처리할수록 연산량이 기하급수적으로 증가하므로, 대용량 VRAM 확보 및 vLLM 같은 추론 최적화 프레임워크 구축 등 물리적 하드웨어 인프라 제약이 따릅니다.

08종합 의견

Qwen2.5-7B-Instruct 모델은 7B 파라미터 체급 대비 긴 문맥 파악 및 지시 이행 능력이 입증되어 RAG 시스템 구축에 매우 유리합니다. 하지만 시각적 데이터가 많은 PPTX 문서의 특성상 모델 자체의 추론 성능보다는 입력되는 텍스트의 구조화 품질이 성패를 좌우합니다. 따라서 즉각적인 전사 도입보다는, 복잡한 레이아웃 문서를 마크다운으로 변환해 주는 '문서 파서(Parser)'와의 연동 테스트를 선행하여 데이터 전처리 파이프라인의 완성도부터 검증하는 기술적 POC 절차를 권고합니다.
